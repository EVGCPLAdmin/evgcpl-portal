How this page works. It calls the portal's Apps Script with each sheet ID, server-side. Apps Script does a real anonymous fetch (no CORS) and returns the actual HTTP status, content-type, and redirect target. That gives a definitive answer to "is this sheet really public?" — bypassing the limitations of browser-side checks.
If the bulk diagnostic doesn't return results, the diagnoseAllSheets_ handler isn't deployed to your Apps Script yet. Paste the snippet from /apps-script/SheetDiagnostic.gs into your project, redeploy, and refresh.

Common verdicts & what they mean

VerdictWhat it meansHow to fix
PUBLIC_OKSheet returned CSV anonymously.No action needed.
NOT_PUBLIC_REDIRECTS_TO_LOGINAnonymous request was bounced to a Google sign-in page.Either set Share → Anyone with the link, OR your Workspace org policy is blocking it (most common).
WORKSPACE_POLICY_BLOCKEDSheet redirected, but to a Workspace policy page.Workspace admin policy is overriding sharing. Allowlist the file at admin.google.com, or move it to a personal account.
TAB_NOT_FOUNDSheet is public, but the requested tab name doesn't exist.Check tab spelling/case in the sheet vs the portal code.
SHEET_NOT_FOUNDThe sheet ID doesn't exist (404).The ID in api.js → SHEETS is wrong, or the sheet was deleted.
